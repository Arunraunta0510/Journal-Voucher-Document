JV Tool Documentation

Integrated Payroll & Expense Journal Vouchers

Overview

The Journal Voucher (JV) tool in HROne automates the preparation of JVs from payroll and expense modules, eliminating the need for manual file preparation. With ready-to-use downloadable files or direct API integration, organizations can close their books faster, reduce manual errors, and ensure compliance with accounting standards.

Supported JV Types

  • Salary JV
  • Salary Advance (Loan) JV
  • Settlement (FNF) JV
  • Expense JV
  • Expense Advance JV

Grouping Options

GL code grouping options:

  • Component wise
  • Business unit wise
  • Department wise
  • Location wise
  • Employee wise
  • Custom grouping available at ₹15K per man-day

Output Formats

  • Excel
  • CSV
  • JSON
  • API for ERP integration
  • FTP drop
  • Any additional/custom output format is available at ₹15K per man-day

Note: You can either manually export the data, drop on a FTP location, or consume HROne API to fetch the data. There is no database-to-database connection support available for security reasons.

    • JV Tool enablement
    • Functional consultant support for configuration

    Note: To get started, clients need to pay a one-time activation cost which covers one time enablement fee and functional support to design the JV. This ensures smooth adoption and readiness to use the JV tool without additional overhead.

  • For Tally integration:
    • Tally connector available through a third-party partner

    Note: For clients using Tally ERP, a certified third-party partner provides a connector that enables direct posting of JV data. In addition to the HROne JV activation fee, clients must pay the partner fee for configuring the JV and establishing the Tally connection.

Integration with ERP

Supported ERPs:

  • SAP
  • Oracle NetSuite
  • Microsoft Dynamics 365
  • TallyPrime (via partner connector)
  • Zoho ERP

Integration Options:

  • Push data via HROne API
  • ERP fetches JV data from HROne API

Note: For advanced requirements such as integrating new financial data categories, designing complex grouping logic, or enabling automated FTP/API push to external systems, HROne provides additional support on a paid basis. This ensures flexibility for unique customer needs.

Not Supported

  • No provisional accounting
  • No direct database posting
  • No cost-splitting of an employee across multiple cost centers

Note: Some use cases are not supported within the JV tool’s standard or extended capabilities. These include provisional accounting entries, direct database postings (due to data security reasons), and allocation of a single employee’s cost across multiple cost centers based on percentages.
